# 应用案例：Harness 与 Loop 如何一起工作

本目录用具体场景说明：Harness 负责控制和验证，Loop 负责持续推进任务。

## 1. 客服知识库自动更新

### 目标

从客服工单中发现高频问题，生成知识库草稿，但不直接发布。

### Harness 重点

- RAG 只接入产品文档、已关闭工单、release notes。
- 输出必须包含来源链接。
- PII 自动脱敏。
- 只能写草稿，不能发布到生产知识库。

### Loop 重点

```text
定时触发
  -> 读取过去 24 小时已关闭工单
  -> 聚类高频问题
  -> 检查现有知识库是否已覆盖
  -> 生成 FAQ 草稿
  -> Guardrail 检查
  -> 通知客服负责人审核
```

### 适用指标

- 重复工单减少率
- 草稿被采纳率
- PII 拦截次数
- 人工审核耗时

## 2. CI 失败自动修复

### 目标

当 CI 失败且 issue 标记为 `agent-ready` 时，Agent 自动复现、修复、测试并创建 PR。

### Harness 重点

- 工作区隔离：使用分支或 worktree。
- 工具限制：只能读写仓库文件、运行测试、查看 diff。
- 保护路径：禁止修改 `.env`、生产部署配置、密钥文件。
- 验证：测试、lint、type check、diff 范围检查。

### Loop 重点

```text
CI failure webhook
  -> 读取失败日志
  -> 复现失败
  -> 定位相关文件
  -> 最小修改
  -> 运行测试
  -> review agent 检查
  -> 创建 PR 或写调查报告
```

### 升级人工的条件

- 需要数据库迁移
- 修改认证、支付、安全相关代码
- 测试无法复现
- 连续两次修复失败

## 3. 工业设备异常诊断

### 目标

监控系统发现异常后，AI 辅助诊断并生成工单草稿，但不直接控制设备。

### Harness 重点

- 传感器数据只读。
- PLC/设备控制权限禁用。
- 必须引用传感器趋势和维修记录。
- 涉及物理动作必须人工批准。

### Loop 重点

```text
异常告警
  -> 读取最近 6 小时传感器数据
  -> 对比历史基线
  -> 检索维修记录
  -> 查询安全 runbook
  -> 可选：数字孪生仿真
  -> 生成工单草稿
  -> 超过安全阈值时升级值班工程师
```

### 升级人工的条件

- 温度、压力、振动超过安全阈值
- 多个传感器结论冲突
- Agent 建议改变设备物理状态

## 4. 研究资料自动整理

### 目标

每天追踪某技术方向的新论文、官方博客和 release notes，生成带来源的研究摘要。

### Harness 重点

- 来源分级：论文、官方文档、release notes 优先。
- 事实、观点、假设、待验证问题分开写。
- 每条关键事实标注日期和来源。
- 不把未验证观点写成结论。

### Loop 重点

```text
每日定时触发
  -> 搜索新资料
  -> 去重和来源分级
  -> 摘要关键贡献
  -> 抽取开放问题
  -> 写入日报草稿
  -> 记录未验证来源
```

### 适用指标

- 来源覆盖率
- 摘要被引用次数
- 未来源化事实数量
- 人工修订比例

## 5. 选型建议

| 场景 | 先强化 Harness | 先强化 Loop |
|---|---|---|
| 客服、法务、金融、医疗 | 是，防幻觉和越权优先 | 否，先不要全自动 |
| 代码维护、测试修复 | 是，工具权限和验证优先 | 是，适合自动循环 |
| 工业运维 | 是，安全边界优先 | 是，但只做诊断和工单 |
| 研究资料 | 是，来源与引用优先 | 是，适合定时整理 |
| 营销内容生成 | 中等，品牌和合规护栏 | 中等，适合草稿工作流 |

经验法则：如果结果会直接影响客户、生产、安全或资金，先把 Harness 做厚；如果任务重复、可验证、可回滚，再逐步加 Loop 自动化。
